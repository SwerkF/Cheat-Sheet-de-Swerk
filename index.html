>_ Link: Initialiser sa balise <head>.

Il est tout le temps difficile de savoir ce qu'il faut mettre dans la balise <head> de son code. Je vous propose un début et une bonne organisation:

                    
        <head>
        <!-- Links -->
            <link rel="stylesheet" href="style.css">

            <!-- Scripts -->
            <script src="main.js" type="module"></script>

             <!-- Meta Data -->
            <meta charset="utf-8">
            <meta name="title" content="Website Title"/>
            <meta name="description" content="Website Description"/>
            <meta name="viewport" content="width=device-width, initial-scale=1">
        </head>

>_ Métadonnées:

Concrétement, la vrai question c'est: "Que est ce c'est des métadonnées?".

Ce sont des balises HTML qui permettent de décrire votre site aux utilisateurs et les moteurs de recherches. Ces données peuvent influencer le référencement de votre site et peuvent être mal interprété par d'autres outils.

Il est donc important d'utiliser ces données afin de promouvoir le site. Ces données peuvent être utilisé pour afficher le nom de la page, la description mais aussi la langue.

Titre:

                    
    <meta property="swerk:title" content="Cheat Sheet - Swerk" />

Description:

                    
    <meta property="swerk:description" content="Il est temps de gagner du temps. Bienvenue sur le Cheat Sheet de Swerk." />

Situer:

                        
    <meta property="swerk:locale:alternate" content="fr_FR" />

Le "swerk:" est la page à laquelle on se réfère. Vous placer un prefix dans votre balise html comme ceci : <html prefix="swerk" />


Voici ce qui s'affiche sur les navigateurs et applications.

>_ Frameworks:

Il ne faut pas confondre Framework et Librairies

.

Un Framework permet d'executer le code que l'utilisateur fourni et la librairie c'est l'utilisateur qui execute le code que elle fourni.

Un Framework fournit plusieurs éléments, ici dans le cas en HTML, des fichiers CSS prêt à être utilisé. Le site sur lequel vous êtes est fait avec le Framework Bootstrap.

Voici une petite liste des Frameworks connu:

>_ Balises méconnues:

Dans le vaste monde qu'est le HTML, il existe aussi quelques balises méconnues. Elles peuvent servir de temps à autres et en voici:

<q>:
Je prend un short bleu ou rouge pour la plage?

Cette balise permet de faire des citations rajoutant automatiquement les guillements

<mark>:

Je vous le vends pour 100€.

Cette balise permet de mettre en exposition le texte. Il suffit d'entourer le mot avec la balise.

<details>:
-> Cliquez moi <-

Bouh!

Voici le code pour le faire:

                        
            <details>
                <summary> -> Cliquez moi <- </summary>
                <p> Bouh! </p>
            </details>

Cette balise permet de faire apparaitre du texte. Plutôt pratique, on est pas obligé de passer par du JavaScript.

>_ GetWaves

GetWaves.io est un site permettant de générer des vagues au format SVG.

Sur ce site, nous pouvons modifier l'intensité, la couleur, la forme et la direction. Il suffit juste d'insérer l'image svg dans notre code.

Il suffit de cliquer sur le petit nuage pour télécharger l'image.

>_ Coolors

>_ SansJQuery

SansJQuery est un site qui permet de regrouper des méthodes afin d'éviter l'utilisation de JQuery.

Ce language n'est plus forcément d'actualité, il faut donc s'orienter vers ce site ou alors développer soit même les fonctions de JQuery en JavaScript.

Exemple tiré du site, effet Fade In:

JQuery: 
                                    
        $(element).fadeIn(200);
JavaScript + CSS: 
                                    
        element.classList.add('show');
        element.classList.remove('hide');
                                
                                    
        .show {
            opacity: 1;
            transition: opacity 500ms;
        }
        .hide {
            opacity: 0;
        }

>_ Symfony

Symfony est un framework qui représente les librairies PHP qui peut être utilisé pour des sites webs dynamique de manière rapide et structurée. Sa facilité de prise en main permet au développeur de travailler facilement dessus.

Comment installer Symfony? (A la date du 10/09/2022)

Pour commencer, il vous faut PHP en version supérieur à 7.2 et il faut installer Scoop, un command-liner pour Windows. Il permet d'installer des programmes en ligne de commande.

Avec PowerShell :

                        
        Set-ExecutionPolicy RemoteSigned -Scope CurrentUser # Needed to run a remote script the first time
        irm get.scoop.sh | iex
        scoop install symfony-cli

Il vous faudra aussi Composer, disponible ICI.

> Démarrer un projet Symfony

Initialiser le projet:

Avant tout, il faut initialiser le projet. Ouvrez votre invité de commande et rentrez cette commande:

                            
        composer create-project symfony/skeleton nomDuProjet

Installer les dépendences et les initialiser:

                            
        composer require make 
        composer require annotations
        composer require twig
        composer require symfony/orm-pack:^2.1

Dans votre Visual Studio Code, ouvrez un nouveau terminal et rentrez cette commande:

                            
                                php bin/console make:controller

Démarrer un server:

                            
        php bin/console make:controller

Créer un controller:

                            
        symfony serve:start
